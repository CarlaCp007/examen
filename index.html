50 Términos Fundamentales del Desarrollo Web

# Término Definición
1HTMLLenguaje de marcado utilizado para estructurar contenido en la web.
3JavaScriptLenguaje de programación que permite añadir interactividad a sitios web.
2CSSLenguaje usado para definir el estilo visual de documentos HTML.
5POSTMétodo HTTP que envía datos al servidor en el cuerpo de la solicitud.
4GETMétodo HTTP que solicita datos del servidor a través de la URL.
6PHPLenguaje de programación del lado del servidor usado para generar contenido dinámico.
7isset()Función de PHP que verifica si una variable está definida y no es null.
8ServidorComputadora o sistema que responde a peticiones web y entrega recursos.
9HostNombre o dirección IP que identifica un dispositivo en la red.
10DominioNombre legible que representa una dirección web (por ejemplo, google.com).
11URLDirección que indica la ubicación de un recurso en la web.
12HTTPProtocolo de transferencia utilizado para comunicar cliente y servidor web.
13HTTPSVersión segura del protocolo HTTP que usa cifrado SSL/TLS.
14FrontendParte del desarrollo web que se ejecuta en el navegador del usuario.
15BackendParte del desarrollo web que se ejecuta en el servidor y procesa lógica y datos.
16Base de datosSistema organizado para almacenar, gestionar y recuperar información.
17MySQLSistema de gestión de bases de datos relacional muy usado en aplicaciones web.
18APIInterfaz que permite a aplicaciones comunicarse entre sí.
19FrameworkConjunto de herramientas y bibliotecas para facilitar el desarrollo web.
20LaravelFramework de PHP para desarrollo de aplicaciones web robustas.
21ReactBiblioteca de JavaScript para construir interfaces de usuario.
22AngularFramework de JavaScript para aplicaciones web SPA (Single Page Application).
23Vue.jsFramework progresivo de JavaScript para construir interfaces interactivas.
24JSONFormato de texto ligero para el intercambio de datos estructurados.
25AJAXTécnica para enviar y recibir datos del servidor sin recargar la página.
26DOMInterfaz de programación que representa la estructura del documento HTML.
27BootstrapFramework CSS para diseñar sitios web responsivos rápidamente.
28ResponsivoDiseño web que se adapta a diferentes tamaños de pantalla y dispositivos.
29Media QueryRegla CSS que aplica estilos según el tamaño del dispositivo.
30SEOOptimización de sitios web para mejorar su visibilidad en buscadores.
31CMSSistema para crear y gestionar contenido web sin necesidad de programar (como WordPress).
32WordPressCMS popular para creación de blogs y sitios web dinámicos.
33CookiePequeño archivo que el servidor guarda en el navegador del usuario.
34SessionMecanismo para guardar datos temporales entre peticiones del usuario.
35TokenClave generada para autenticar y autorizar a un usuario en la web.
36GitSistema de control de versiones para llevar seguimiento de cambios en el código.
37GitHubPlataforma para alojar y colaborar en proyectos de desarrollo usando Git.
38CommitAcción de guardar cambios en el historial de un repositorio Git.
39PushEnviar los cambios locales de Git a un repositorio remoto.
40CLIInterfaz basada en comandos para interactuar con el sistema operativo o herramientas.
41Node.jsEntorno de ejecución para JavaScript en el servidor.
42NPMGestor de paquetes para proyectos en Node.js.
43RESTEstilo de arquitectura de APIs que usa HTTP para acceder a recursos.
44CRUDConjunto de operaciones básicas sobre una base de datos: Crear, Leer, Actualizar y Eliminar.
45XAMPPPaquete que incluye Apache, MySQL, PHP y Perl para desarrollo local.
46WAMPPaquete para Windows que incluye Apache, MySQL y PHP.
47DeploymentProceso de publicar una aplicación web en un servidor.
48WebSocketProtocolo que permite comunicación bidireccional entre cliente y servidor.
49SASSPreprocesador CSS que permite usar variables, funciones y anidamiento en hojas de estilo.
50ViewportÁrea visible de una página web en el dispositivo del usuario.
